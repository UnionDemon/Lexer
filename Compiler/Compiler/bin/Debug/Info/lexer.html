Техническое задание.

Цель работы: Изучить назначение лексического анализатора. Спроектировать алгоритмы и выполнить программную реализацию сканера.

Задание: Разработать универсальный лексический анализатор, позволяющий выделить в тексте типовые лексемы ("число", "идентификатор", "знак", "скобки" и др.).

Предусмотреть перечень недопустимых (не входящих в алфавит) символов.

Входные данные - строка (текст).

Выходные данные - последовательность условных кодов, описывающих структуру разбираемого текста с указанием места положения и типа ("число", "идентификатор", "знак", "недопустимый символ" и т.д.).

Например, для строки "int x=123;":

14 - ключевое слово - int - с 1 по 3 символ

11 - разделитель - (пробел) - с 4 по 4 символ

2 - идентификатор - x - с 5 по 5 символ

10 - оператор присваивания - = - с 6 по 6 символ

1 - целое без знака - 123 - с 7 по 9 символ

16 - конец оператора - ; - с 10 по 10 символ

Встроить сканер в ранее разработанный интерфейс текстового редактора.

Учесть, что текст для разбора может состоять из множества строк.

Встроить в приложение справку с информацией по заданию.


Диаграмма состояний сканера.

Разработан сканер для задачи "Объявление переменных языка C++".

Сканер распознает следующие лексемы:

  1. Ключевое слово - int;
  2. Ключевое слово - bool;
  3. Ключевое слово - char;
  4. Ключевое слово - string;
  5. Идентификатор;
  6. Ключевое слово - true;
  7. Ключевое слово - false;
  8. Разделитель - пробел;
  9. Оператор присваивания;
  10. Строка;
  11. Символ;
  12. Целое без знака;
  13. Конец оператора;
  14. Конец файла;
  15. Недопустимый символ.
